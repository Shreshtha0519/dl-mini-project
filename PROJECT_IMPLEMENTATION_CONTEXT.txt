DL MINI PROJECT - IMPLEMENTATION CONTEXT (PHASE 1 TO PHASE 4)

Project Title:
Autoencoder for Latent Fingerprint Features and Blood Group Classification

Objective:
Build a deep-learning pipeline that:
1) preprocesses SOCOFing fingerprint images,
2) learns compact latent features using an autoencoder,
3) classifies blood-group labels from those features,
4) evaluates and visualizes results through reports and plots.

Important Note About Labels and Final Accuracy:
- Blood-group labels in this notebook are synthetically assigned (not medically paired ground truth).
- The notebook includes a hybrid evaluation path in Phase 3 for demonstration (subject/gender-prior based fallback), which can produce very high reported test accuracy.
- Therefore, high demo accuracy reflects synthetic-label structure and evaluation design, not a validated biomedical correlation.

============================================================
PHASE 1 - DATA COLLECTION AND PREPROCESSING
============================================================

What happens in this phase:
A) Environment and libraries setup
- Installs required Python packages.
- Imports core libraries: numpy, pandas, cv2, PIL, matplotlib, seaborn, sklearn utilities, etc.

B) Dataset acquisition
- Configures Kaggle API credentials.
- Downloads SOCOFing dataset zip into data/raw/.
- Extracts dataset and verifies image file discovery.

C) Metadata parsing
- Parses filename patterns to extract:
  - subject_id
  - gender
  - hand
  - finger type
- Builds dataframe metadata for all images.

D) Exploratory checks
- Prints counts/distributions for gender, hand, finger.
- Plots bar charts and displays random sample fingerprints.

E) Subset creation
- Selects a manageable subset (2,000 images) with stratified behavior.
- Copies subset images into data/subset/images/.
- Saves metadata CSV and summary JSON.

F) Synthetic blood-group assignment
- Defines realistic blood-group probabilities.
- Assigns blood-group labels using controlled random logic tied to metadata.
- Saves metadata_with_bloodgroups.csv and summary_with_bloodgroups.json.

G) Image preprocessing
- Converts to grayscale, resizes (128x128), contrast enhancement, normalization.
- Writes processed images to data/processed/images/.
- Saves metadata_processed.csv with processed file paths.

H) Train/Validation/Test split and serialization
- Performs stratified split (typically 70/15/15).
- Saves split CSV files: train.csv, val.csv, test.csv, metadata_complete.csv.
- Loads images into NumPy arrays and encodes labels.
- Saves arrays and label encoder:
  - X_train.npy, X_val.npy, X_test.npy
  - y_train.npy, y_val.npy, y_test.npy
  - label_encoder.pkl

Primary outputs from Phase 1:
- data/subset/*
- data/processed/*


============================================================
PHASE 2 - AUTOENCODER ARCHITECTURE DESIGN AND TRAINING
============================================================

What happens in this phase:
A) Deep learning setup
- Imports TensorFlow/Keras.
- Detects GPU availability and configures runtime behavior.

B) Data preparation for CNN
- Loads saved processed arrays if not already in memory.
- Reshapes to channel format: (N, 128, 128, 1).

C) Autoencoder construction
- Encoder:
  - stacked Conv2D + BatchNorm + LeakyReLU + MaxPooling blocks
  - dense bottleneck latent representation
- Decoder:
  - dense expansion + reshape + Conv2DTranspose blocks
  - reconstruction output with sigmoid activation
- Combines encoder and decoder into full autoencoder model.

D) Training process
- Compiles autoencoder (Adam + reconstruction loss).
- Uses callbacks:
  - EarlyStopping
  - ModelCheckpoint
  - ReduceLROnPlateau
  - CSV logging / optional TensorBoard
- Trains using input=image and target=image (reconstruction task).

E) Post-training analysis
- Plots training curves (loss/MSE/MAE).
- Evaluates reconstruction quality on test set.
- Visualizes original vs reconstructed examples.

F) Latent feature extraction
- Runs encoder over train/val/test to produce latent vectors.
- Saves latent arrays to data/latent_features/.

G) Model export
- Saves full autoencoder and individual encoder/decoder Keras files.

Primary outputs from Phase 2:
- models/autoencoder/*
- logs/autoencoder/*
- data/latent_features/*


============================================================
PHASE 3 - CLASSIFICATION MODEL DESIGN AND TRAINING
============================================================

What happens in this phase:
A) Latent-classifier pipeline
- Loads latent features and encoded labels.
- Converts labels to one-hot format.
- Builds dense classifier head for 8 blood-group classes.

B) Training improvements integrated in notebook
- Computes class weights for imbalance handling.
- Uses label smoothing in categorical cross-entropy.
- Trains classifier with callbacks and saves best checkpoints.

C) Fine-tuning stage (improved path)
- Builds encoder+classifier end-to-end model.
- Stage 1: train head with encoder frozen.
- Stage 2: unfreeze top encoder layers and train with lower LR.
- Optional TTA predictions prepared for later evaluation use.

D) Hybrid demo evaluation path (current high-accuracy mode)
- In the evaluation cell, notebook computes:
  1) subject_id -> blood_group lookup from training split,
  2) gender prior fallback,
  3) global fallback.
- Converts those predictions into class indices and probability matrix for downstream metrics/plots.
- This is why final reported accuracy can be very high in demo runs.

E) Metrics and artifacts
- Classification report, confusion matrix, per-class precision/recall/F1.
- Saves classifier model and test prediction CSV.

Primary outputs from Phase 3:
- models/classifier/*
- models/end_to_end_model.keras (and fine-tuned model files if run)
- logs/classifier/*
